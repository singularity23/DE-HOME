JS — scheduling & progress
perf▶
What this does in plain English: 
on radio change / DOM mutation:
  scheduleUpdateProgress()  // max 1× per animation frame
    → count checked radios per cached rowsEl (not document-wide query)
    → update badges, % bar
    → if 100% → allRequiredFieldsValid() for Print/PDF button

on comment input:
  scheduleSyncCommentsHeights()  // pairs checker/engineer comment boxes
  • Good: Avoids layout thrash when typing fast in Quill/fallback editors.
  • Watch: allRequiredFieldsValid() still runs getComputedStyle on every visible required field at 100% — acceptable but hot on large forms.
JS — comment editors (Quill + fallback)
resilience▶
Review note: Beta’s biggest behavioral addition. If Quill fails, users get a contenteditable box with Ctrl+V screenshot support and hidden-field sync — critical for engineers opening saved snapshots from disk.
